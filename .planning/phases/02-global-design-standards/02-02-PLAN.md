Add a max-width wrapper inside the page content area (per D-14).

In PageShell.tsx, wrap `{children}` inside the `.page-shell-content` div with a constraining div:

Change the content section from:
```tsx
<div className="page-shell-content" style={{ padding: '24px 40px 48px', flex: 1 }}>
  {children}
</div>
```

To:
```tsx
<div className="page-shell-content" style={{ padding: '24px 40px 48px', flex: 1 }}>
  <div className="max-w-[1280px] mx-auto">
    {children}
  </div>
</div>
```

Also apply the same constraint to the header section. Wrap the header flex container:

Change the header from:
```tsx
<div className="page-shell-header" style={{ padding: '32px 40px 0', display: 'flex', ... }}>
```

To a two-level structure:
```tsx
<div className="page-shell-header" style={{ padding: '32px 40px 0' }}>
  <div className="max-w-[1280px] mx-auto" style={{ display: 'flex', alignItems: 'flex-start', justifyContent: 'space-between', gap: 16 }}>
    ...title, subtitle, actions...
  </div>
</div>
```

This ensures both header and content align at the same 1280px max-width. On screens under 1280px, no visual change. On screens over 1280px (like a 1440px display), content centers with balanced margins.

Do NOT change the mobile responsive overrides in the `<style>` block.
