### Border Radius -- Global CSS Variable Fix
The current `--radius` is `0.625rem` (10px). Halving to `0.3125rem` (5px) via a single CSS variable change in `src/styles/globals.css` will propagate through all Tailwind `rounded-*` classes and shadcn components because the tailwind config calculates all radius values from `var(--radius)`.

Additionally update the explicit radius variables:
```
--radius: 0.3125rem      (was 0.625rem = 10px, now 5px)
--radius-sm: 3px          (was 6px)
--radius-md: 5px          (was 10px)
--radius-lg: 7px          (was 14px)
--radius-xl: 9px          (was 18px)
--radius-pill: 100px      (keep -- pills should stay round)
```

Any component using hardcoded `rounded-[14px]` or similar must be audited and switched to use the token system.

### Claude API Parsing -- Blocker Debug
The parsing function in `src/hooks/useBrainDump.ts` (line 22-80) calls the Anthropic API directly via `fetch`. The error "Could not parse response from Claude" comes from the JSON parse catch block (line 74-79). Root causes to check:

1. **Claude returns markdown-wrapped JSON:** Claude often wraps JSON in ` ```json ``` ` blocks. The parser does `JSON.parse(content)` directly without stripping markdown fencing. Fix: strip ` ```json ` and ` ``` ` before parsing.
2. **API key missing/invalid:** Check if `VITE_ANTHROPIC_API_KEY` is set in `.env.local`. If missing, it falls back to local parsing, which should work. If set but invalid, it throws on line 66.
3. **Model name:** Using `claude-sonnet-4-20250514` -- verify this model ID is current and accessible.
4. **CORS issues:** The `anthropic-dangerous-direct-browser-access` header is set, which is correct for browser-side calls.

**Most likely fix:** Add JSON extraction that strips markdown fencing before `JSON.parse`:
```typescript
// Strip markdown code fencing if present
let jsonStr = content.trim()
if (jsonStr.startsWith('```')) {
  jsonStr = jsonStr.replace(/^```(?:json)?\s*/, '').replace(/\s*```$/, '')
}
return JSON.parse(jsonStr) as ParsedBrainDump
```

## Common Pitfalls

### Pitfall 1: Claude JSON Wrapping
**What goes wrong:** Claude wraps JSON responses in markdown code fencing (` ```json ... ``` `) even when instructed not to.
**Why it happens:** LLM instruction following is probabilistic; "Return JSON only, no markdown" fails sometimes.
**How to avoid:** Always strip markdown fencing before JSON.parse. Add a regex extraction step.
**Warning signs:** "Could not parse response from Claude" error message.

### Pitfall 2: Tailwind Purge Breaking Dynamic Classes
**What goes wrong:** Dynamic Tailwind classes like `border-l-[${color}]` get purged in production builds.
**Why it happens:** Tailwind scans for complete class strings at build time; string interpolation creates classes it cannot find.
**How to avoid:** Use the static class mapping pattern already established: `const borderClasses = { success: 'border-l-[hsl(var(--status-success))]' }`.
**Warning signs:** Styles work in dev but disappear in production build.

### Pitfall 3: Position Sticky Not Working
**What goes wrong:** `position: sticky` headers don't stick.
**Why it happens:** Parent elements have `overflow: hidden` or `overflow: auto` that creates a new stacking context, or no explicit `top` value.
**How to avoid:** Always set `top: 0` (or appropriate offset). Check that no ancestor has `overflow: hidden`. The sticky element must be a direct child of a scrollable container.
**Warning signs:** Headers scroll with content instead of sticking.

### Pitfall 4: Brand Icon Import Size
**What goes wrong:** Importing all icons from `@icons-pack/react-simple-icons` bloats the bundle.
**Why it happens:** Barrel imports pull in the entire library.
**How to avoid:** Import individual icons: `import { SiFacebook } from '@icons-pack/react-simple-icons'`. The library is tree-shakeable with individual imports.
**Warning signs:** Chunk size warning in Vite build (already present but manageable).

### Pitfall 5: Clipboard API Permission
**What goes wrong:** `navigator.clipboard.writeText()` fails silently or throws.
**Why it happens:** Clipboard API requires secure context (HTTPS or localhost) and sometimes user gesture.
**How to avoid:** Always wrap in try/catch, show error toast on failure. Dev server runs on localhost so it works.
**Warning signs:** Copy button does nothing with no error visible.

### Pitfall 6: CSS Grid on Stat Tiles -- Breakpoint Mismatch
**What goes wrong:** 5 equal columns work on wide screens but overflow on tablet.
**Why it happens:** 5 columns at 768px is too narrow per column.
**How to avoid:** Use `grid-template-columns: repeat(5, 1fr)` for desktop, `repeat(2, 1fr)` for mobile, with the 5th tile spanning full width or adapting.
**Warning signs:** Tiles truncate or overflow on smaller screens.
