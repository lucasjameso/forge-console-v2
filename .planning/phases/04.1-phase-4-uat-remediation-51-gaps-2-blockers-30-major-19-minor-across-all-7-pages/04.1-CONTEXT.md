<decisions>
## Implementation Decisions

### Global (applies to all pages)
- Reduce border-radius globally to approximately half current value on all cards, tiles, badges, containers, input areas, pills, and modals. Clean and modern, not bubbly, not squared off. [BLOCKER-ADJACENT: reported on 6/7 pages]

### Dashboard -- Stat Tiles
- Stat tiles must use CSS grid with equal fr columns, not flexbox. All 5 tiles maintain equal sizing across all viewports (55-inch monitor to laptop). CLARITY Launch tile must not be different size.
- Move recency colored borders from top edge to left edge of each stat tile. Left-side accent borders are the standard pattern.
- CLARITY countdown pulsing animation must activate below 14 days. Amber static at 14-30 days is correct. Verify pulsing is wired up.

### Dashboard -- Action Items
- Action items must be interactive: clicking opens detail view or modal for taking action (mark complete, reassign, add notes, snooze).
- Replace card-style layout with compact scannable list format. Subtle separators, not chunky card blocks. Scan 20 items quickly without excessive padding.

### Dashboard -- Content Calendar
- Content calendar cards: add slightly more height so full titles display without ellipsis.
- Content calendar cards: show content type (carousel, text post, article, etc.) below the title.

### Dashboard -- Greeting
- Greeting subtitle should rotate through high-value contextual insights dynamically: "CLARITY launches in 25 days", "Meridian has had no activity in 4 days", "3 content pieces need approval this week". Pick most urgent data point, not just action item count.

### Brain Dump -- Core Function (BLOCKER)
- Claude API parsing broken: submitting text returns "Could not parse response from Claude." Debug API call -- check Anthropic SDK config, API key, response handling. CRITICAL.
- "+ Add to project" button on parsed task cards must open dropdown or modal to select project. Currently non-functional.
- Submit button visual state: empty = muted/disabled appearance. Text entered = glowing coral with subtle pulse or brightness increase.
- Replace pill-style status indicators with horizontal step progress indicator. Connected steps with arrows or progress line. Completed=filled, active=pulsing, future=grayed. Pipeline, not tag collection.
- Parsed task cards need inline action controls: assign to project dropdown, change priority dropdown, edit task text inline, accept/reject parsed task, re-parse button. Small icon buttons and small dropdowns.

### Brain Dump -- History
- History entries must use project-specific left border colors (Ridgeline color, Forge color, etc.), not uniform coral. Left border is primary visual cue for project scanning.
- Day group headers (Yesterday, March 20) must be position:sticky with background color when scrolling.

### Content Pipeline -- Review Modal (BLOCKER)
- Content review modal must match W4_Batch.html quality: full post body text (scrollable/selectable), character count with optimal range indicator (green 1200-1600, amber outside), Copy Post Text button with LinkedIn-safe formatting, Copy Without Hashtags button, content type badge, carousel slide breakdown with titles/descriptions/source paths, thumbnail previews with slideshow navigation, platform metadata.
- Reject workflow: open rejection reason text input, log reason to Supabase, show reason when viewed later for audit trail.
- Approve workflow: mark approved, surface in Ready to Post queue, show Copy Post button immediately.

### Content Pipeline -- Grid and Forms
- Calendar grid must adapt fluidly to different screen widths (55-inch to laptop) without overlapping or unreadable truncation.
- Add Content modal redesign: rename Caption to "Post Body", full-height textarea (min 300px) with auto-grow, live char count (green 1200-1600, amber outside, red >3000), Content Type selector (Text/Carousel/Visual Quote/Poll), fix platforms (replace Medium/Goodreads/Amazon with Facebook/X/Instagram/TikTok, multi-select), auto-calc Week Number from date, Series/Arc tag field, Notes field.
- Kanban cards must be draggable between status columns (Draft > Pending Review > Approved > Posted) with timestamp logging.
- Multi-platform support: LinkedIn primary with variant generation for FB/X/IG/TikTok. Data model needs parent post + child variant records. At minimum show platform selector and multi-platform tags.
- Detail modal must open identically from all 4 views (List, Week, Month, Kanban).

### Social Media
- All platforms must show real brand icons from @icons-pack/react-simple-icons. Zero generic globe icons.
- Medium account data: verify @lucasoliver/240 followers is accurate or correct/remove.
- All platform cards same size regardless of active/setup status. Setup-needed platforms (Amazon/Goodreads) get equal visual weight.
- "Setup Needed" becomes actionable button opening modal with: account to create, handle to reserve, credentials to connect, pre-launch checklist.
- Active platform cards show: follower count, follower trend, last post date, posts this month, engagement rate placeholder. Every platform with follower goal gets progress bar.
- Hero stat row: add "Platforms ready for CLARITY launch: 2/13" progress indicator.
- All platform cards (active and setup) have external link icon to profile URL or account creation page.

### Settings
- Integration test buttons: n8n (health endpoint ping), Claude API (minimal completion), Slack (webhook ping), Cloudflare (Pages API). If not implementable, hide button and show muted "Test coming soon".
- Feedback expanded entries: proper markdown rendering with visual hierarchy, separated numbered items, bold labels, indentation. Not flat text walls.
- Feedback entries need action controls: mark Done, assign to GSD phase, add note, mark Won't Fix with reason.
- System tab: add last deploy timestamp, Supabase table/row counts, API call usage, system uptime.
- Preferences and Data Management tabs: show "Coming soon" empty state when clicked, or hide entirely.
- Brand icons: verify n8n, Slack, Claude API show actual brand icons.

### Activity Log
- Density chart: 3-4x taller (min 120px), hover tooltips with day breakdown, color legend below mapping colors to projects.
- Three distinct visual tiers: MAJOR (bordered card, shadow, bold), STANDARD (compact row, badges), BACKGROUND (single line, gray, reduced opacity, no card).
- Entries must be clickable and link to source (project detail, content item, etc.).
- Timeline dot color legend mapping colors to tools.
- Date range filter with presets: Today, Yesterday, Last 7d, Last 30d, Custom range.
- Entry alignment: tool badges, project badges, descriptions, timestamps must snap to consistent grid.
- Manual entry form: add tool dropdown (Claude Code, n8n, Slack, Manual, System), event type dropdown (milestone, commit, approval, note, health check), optional link URL.
- Day group headers must be position:sticky with background color.
- Load More button must actually load additional entries.
